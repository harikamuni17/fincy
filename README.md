## Core Features

### Anomaly Detection Engine
The Analyzer Agent calculates mean, standard deviation, and Z-scores across every vendor, department, and category. Anything with `|Z| > 2.0` is flagged as anomalous ‚ÄĒ not guessed, not approximated. The full calculation steps are stored and displayed for every finding.

```
Finding: AWS Spend Spike ‚ÄĒ Engineering Dept
‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ
Baseline avg (Jan‚ÄďFeb)  ‚āĻ42,000 / month
Current month (Mar)     ‚āĻ1,17,600
Delta                   ‚āĻ75,600 excess
Z-score                 4.1ŌÉ  (statistically impossible to be random)
Confidence              87%
Annual risk             ‚āĻ9,07,200
```

### Four-Agent AI Pipeline

| Agent | Role | Output |
|-------|------|--------|
| **Analyzer** | Statistical anomaly detection across all expense dimensions | `Finding[]` with full math + `AgentReasoning[]` |
| **Decision** | Ranks corrective actions by ROI, assigns approval tier | `ActionLog[]` sorted by monthly saving |
| **Action** | Executes autonomous actions ‚ÄĒ Slack, vendor freeze, ticket | Confirmation receipts + audit trail |
| **Monitor** | Compares predicted vs actual savings after execution | `ROIRecord[]` with accuracy percentage |

### Enterprise Approval Workflow

Actions are automatically routed by dollar impact ‚ÄĒ no manual configuration required.

```
< ‚āĻ500          ‚Üí  AUTO        (executes immediately, no approval needed)
‚āĻ500 ‚Äď ‚āĻ5,000  ‚Üí  MANAGER     (48-hour SLA, email notification sent)
‚āĻ5,000 ‚Äď ‚āĻ20K  ‚Üí  DIRECTOR    (36-hour SLA)
> ‚āĻ20,000       ‚Üí  CFO         (24-hour SLA, escalation if overdue)
```

Every approval requires a documented reason code. Every action generates an immutable audit entry with approver identity and timestamp.

### Autonomous Execution

When an action is approved, Finci fires three things simultaneously:

```bash
‚úď  Slack alert delivered to #finance-alerts
‚úď  Vendor frozen ‚ÄĒ all future transactions flagged
‚úď  Internal ticket FINCI-{ID} created and assigned
```

### Loss Prevention (Forecast Engine)

The Forecast Engine runs linear regression on historical spend data to project the next 30 days. It detects budget breaches before they happen and surfaces prevention actions directly on the forecast chart.

```
‚ö†  ACTIVE ALERT
   At current trajectory, you will exceed AWS committed spend
   by ‚āĻ2.1L in 23 days ‚ÄĒ triggering a ‚āĻ40,000 SLA penalty.
   
   ‚Üí Prevent This  (routes to Action Center)
```

### Scenario Simulator

Before approving any action, the CFO can simulate the full financial impact with interactive sliders. The Chart.js projection updates in real time ‚ÄĒ no server call, no reload.

```

## API Reference

### Data Ingestion

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/ingest/card-feed` | Generate synthetic corporate card transactions |
| `POST` | `/api/ingest/csv` | Upload and parse expense CSV |
| `POST` | `/api/ingest/pdf` | Upload bank statement PDF ‚ÄĒ OpenAI extracts transactions |

### Analysis Pipeline

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/analyze?sessionId=` | Run the 4-agent pipeline ‚ÄĒ streams progress via SSE |
| `GET` | `/api/findings?sessionId=` | List all findings for a session |
| `GET` | `/api/actions?sessionId=` | List all recommended actions |

### Approval Workflow

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/actions/[id]/approve` | Approve action ‚ÄĒ requires `reasonCode` in body |
| `POST` | `/api/actions/[id]/reject` | Reject action ‚ÄĒ requires `reasonCode` in body |
| `POST` | `/api/optimize-batch` | Execute all AUTO-tier actions simultaneously |

### Intelligence

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/chat` | Finci AI chat ‚ÄĒ OpenAI with full session context injected |
| `POST` | `/api/simulate` | Run what-if projection for an action |
| `GET` | `/api/forecast?sessionId=` | Fetch 30-day loss projection and alerts |
| `GET` | `/api/roi?sessionId=` | Predicted vs actual savings comparison |

### Exports

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/export/pdf` | Generate 4-page CFO Summary Report (PDFKit) |
| `POST` | `/api/export/pptx` | Generate 5-slide Board Deck (pptxgenjs) |

```

PolicyViolation  ‚Üí  Rules triggered against live transactions
LiveFeedEvent    ‚Üí  Real-time webhook transaction stream
TicketLog        ‚Üí  Internal FINCI-{ID} tickets per action
User             ‚Üí  Auth users with approval tier roles
```

## What Makes Finci Different

Most expense tools produce a report. Finci runs a loop.

```
Other tools:   Upload ‚Üí Analyze ‚Üí Report

Finci:         Upload ‚Üí Analyze ‚Üí Act ‚Üí Simulate ‚Üí Approve ‚Üí Execute ‚Üí Measure

Every rupee saved is traced back to the specific agent decision, the specific approver, and the specific timestamp ‚ÄĒ creating a complete financial audit trail that enterprise compliance teams can rely on.
